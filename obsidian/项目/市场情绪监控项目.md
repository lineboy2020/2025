# 市场情绪监控项目

## 项目定位

围绕 `qingxu.parquet` 构建可持续演进的 A 股市场情绪监控体系，目标不是只做一个可视化页面，而是形成：

1. **数据层**：稳定、可解释的市场情绪指标源
2. **展示层**：网页图表、情绪评分、周期标签、红绿灯
3. **研究层**：情绪周期定义、阈值迭代、经验回测
4. **记录层**：将项目更新、技术文档、经验沉淀到 Obsidian，形成长期知识库

---

## 当前范围

### 数据来源
- 主数据文件：`data/db/qingxu.parquet`
- 服务入口：`kline-viewer/scripts/kline_server.py`
- 前端页面：`kline-viewer/static/qingxu.html`

### 当前核心字段
- `rise_count` 上涨家数
- `fall_count` 下跌家数
- `limit_up_count` 涨停家数
- `limit_down_count` 跌停家数
- `total_count` 样本总数
- `rise_ratio` 上涨占比
- `fall_ratio` 下跌占比
- `explosion_count` 炸板数
- `explosion_rate` 炸板率

### 当前派生指标
- `emotion_score` 市场情绪评分（0-100）
- `emotion_level` 强/中/弱
- `emotion_label` 强势 / 中性偏强 / 偏弱
- `traffic_light` green / yellow / red
- `phase_tag` 冰点 / 退潮 / 修复 / 过热 / 震荡
- `phase_note` 周期解释说明

### 4. 每日跟踪
- `交易系统/市场情绪每日跟踪模板.md`
- 用于每天盘后持续记录情绪判断、数据特征、次日预期

---

## 持续跟踪建议

后续建议每天至少记录以下内容：

1. **当日情绪结论**
   - 强 / 中 / 弱
   - 冰点 / 修复 / 过热 / 退潮 / 震荡

2. **关键数据**
   - 上涨/下跌家数
   - 涨停/跌停家数
   - 炸板率
   - 上涨/下跌占比

3. **结构观察**
   - 是普涨还是局部热点？
   - 连板高度是否提升？
   - 高位股是加强还是分歧？
   - 是指数带动还是情绪独立修复？

4. **交易层意义**
   - 适合打板 / 低吸 / 趋势跟随 / 防守
   - 次日更适合进攻还是兑现
