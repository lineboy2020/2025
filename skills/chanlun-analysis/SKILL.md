# 缠论分析技能 (Chanlun Analysis Skill)

## 概述

本技能用于把缠论结构输出，稳定转换成**实战可读的判断结果**。重点不是空谈理论，而是围绕以下几个问题给出统一答案：

- 这只股票当前是 **强 / 中 / 弱**？
- 当前是 **上涨趋势 / 下跌趋势 / 中枢震荡**？
- 现在所处位置是 **中枢上方 / 中枢内 / 中枢下方** 还是高低位切换区？
- 当前有没有买点？如果有，是 **1买 / 2买 / 3买** 中哪一种？
- 当前是 **候选 / 待确认 / 已确认 / 失效 / 降级** 中哪种状态？
- 当前是否具备参与性？是 **可试错 / 可低吸 / 可跟随 / 只观察 / 不参与**？
- 结构止损位应该放哪里？

本技能允许基于已有缠论算法输出做解释增强，但**默认不轻易修改缠论核心算法**。如果用户明确要求“先读透，不轻易改算法”，应优先做解释层、决策层、页面/API层增强。

## 当前项目共识（重要）

以下规则是当前技能升级时必须遵守的既定方向：

### 1. 理论层 vs 工程层

#### 理论层
- **1买**：采用分层定义
  - 候选1买
  - 标准1买
- **2买**：理论层不分层
- **3买**：理论层不分层

#### 工程层
允许状态化输出：
- `candidate`
- `observe`
- `waiting_confirm`
- `confirmed`
- `downgraded`
- `invalidated`

### 2. 1买的定义
- 1买不是“任何底分型”
- 必须放在**离开中枢后的下跌末端**理解
- **标准1买必须有背驰**
- 没有明显背驰，但有较强底分型 + 向上一笔，可记为**候选1买**

### 3. 2买的定义
- 必须建立在 1买之后
- 先有向上确认
- 再有回踩
- 回踩不破关键结构
- 再次转强，才进入二买确认逻辑

### 4. 3买的定义
- 必须绑定中枢
- 必须是**离开中枢后的回踩确认**
- 允许影线测试
- 但**不能有效回到中枢内部**

### 5. 确认条件
确认优先级遵循：
1. **先看分型强度**
2. 再看后续结构是否跟随确认

不能把“单根阳线 / 单个收盘站上”直接当成确认。

### 6. 结构止损优先
当前技能升级后，默认采用：
- **1买止损 = 底分型最低点**
- **2买止损 = 1买低点 / 关键回踩结构低点**
- **3买止损 = 中枢边界 / 关键回踩低点**

不要默认在结构止损里混入 95% 之类的工程化缓冲，除非用户明确要求输出“交易止损”。

## 核心功能模块

### 1. K线包含关系处理 (Containment Processing)

处理K线之间的包含关系，遵循缠论标准合并规则。

### 2. 分型识别 (Fractal Detection)

识别顶分型和底分型，包括标准分型和强力分型。

**分型类型**:
- `top`: 标准顶分型
- `strong_top`: 强力顶分型
- `bottom`: 标准底分型
- `strong_bottom`: 强力底分型
- `single_k_top`: 单K反转顶
- `single_k_bottom`: 单K反转底

### 3. 笔识别 (Bi / Stroke Detection)

识别缠论中的“笔”，即连接相邻顶底分型的结构线。

### 4. 中枢识别 (Zhongshu Detection)

识别中枢区间，用于判断：
- 趋势推进
- 中枢震荡
- 中枢离开
- 3买 / 3卖成立与否

### 5. 背驰检测 (Divergence Detection)

用于辅助判断：
- 1买是否具备标准意义上的衰竭证据
- 1卖是否具备标准意义上的上涨衰竭证据

当前项目共识：
- **标准1买必须有背驰**
- 没有背驰可降级为候选1买/底部观察位，但不与标准1买同权

### 6. 买卖点识别 (Buy / Sell Points)

识别：
- `1B` / `2B` / `3B`
- `1S` / `2S` / `3S`

注意：
- `buy_points` / `sell_points` 更适合看作**历史结构信号样本**
- `current_state` 更适合看作**当前实时所处阶段**
- 两者不能混为一谈

### 7. 实战解释层（本技能升级重点）

本技能的重点升级方向不是增加更多术语，而是输出以下实战结论：

- 趋势判断
- 强弱判断
- 所处位置
- 买点模式
- 买点状态
- 可参与性
- 结构止损
- 目标区间
- 风险等级

## 固定输出框架（今后默认采用）

当用户请求“分析一只股票”“判断当前是否有买点”“用缠论看强弱/趋势/位置/参与性”时，优先按以下模板输出：

### 1）趋势
- 上涨趋势 / 下跌趋势 / 中枢震荡

### 2）强弱
- 强 / 中 / 弱

### 3）位置
- 中枢上方 / 中枢内 / 中枢下方
- 高位 / 中位 / 低位

### 4）买点模式
- 无买点 / 1买 / 2买 / 3买
- 若是1买，要区分：
  - 候选1买
  - 标准1买

### 5）状态
- 候选 / 观察 / 待确认 / 已确认 / 降级 / 失效

### 6）参与性
- 可试错
- 可低吸
- 可跟随
- 只观察
- 不参与

### 7）风险锚点
- 结构止损位
- 失效条件

### 8）一句话结论
用 1~2 句话给出交易结论，不要只罗列指标。

## 推荐输出模板（可直接复用）

```markdown
# 缠论分析结论

## 1）趋势
- 结论：上涨趋势 / 下跌趋势 / 中枢震荡
- 依据：最近笔结构 + 中枢关系

## 2）强弱
- 结论：强 / 中 / 弱
- 依据：高低点抬升/下移、分型强弱、是否有效离开中枢

## 3）位置
- 结论：中枢上方 / 中枢内 / 中枢下方
- 补充：高位 / 中位 / 低位

## 4）当前买点
- 结论：无买点 / 候选1买 / 标准1买 / 2买 / 3买
- 状态：观察 / 待确认 / 已确认 / 降级 / 失效

## 5）可参与性
- 结论：可试错 / 可低吸 / 可跟随 / 只观察 / 不参与
- 原因：结构是否完整、是否已确认、是否临近风险区

## 6）风险锚点
- 结构止损位：xx.xx
- 失效条件：跌破xx / 有效回中枢 / 破坏关键结构

## 7）一句话结论
- 当前更接近……，如果……则……，否则……
```

## 页面 / API 输出建议

当该技能用于页面、接口或结构化输出时，建议使用以下字段组织：

### 模式字段
- `setup_type`: `1B` / `2B` / `3B` / `none`
- `setup_type_zh`: `一买` / `二买` / `三买`

### 状态字段
- `setup_stage`: `candidate` / `observe` / `waiting_confirm` / `confirmed` / `downgraded` / `invalidated`
- `setup_stage_zh`: 对应中文状态

### 辅助字段
- `trend_structure`
- `strength`
- `position_zone`
- `trade_permission`
- `position_advice`
- `structural_stop_loss`
- `invalidation_condition`
- `history_stats`

## 历史统计模块使用原则

当接入真实胜率统计时，必须遵守：

1. **只能用真实历史K线 / 真实结构样本统计**
2. **禁止伪造胜率、收益、回撤**
3. 如果统计未接入，必须明确显示：
   - 待统计
   - 未接入
   - 样本不足
4. 后续统计应优先支持：
   - 候选1买 vs 标准1买
   - 2买确认
   - 3买确认
   - 1日 / 3日 / 5日窗口

## 何时适合调用本技能

适用于以下用户请求：
- “用缠论分析一下这只票”
- “这票现在强不强？”
- “是趋势还是震荡？”
- “当前有没有1买/2买/3买？”
- “止损应该放哪里？”
- “帮我从缠论角度判断位置和参与性”

---

## 注意事项

- 不要把所有底分型都叫 1买
- 不要把单次回踩都叫 2买
- 不要把普通趋势回踩都叫 3买
- 不要把“历史买点样本”误说成“当前立刻可买”
- 不要把“结构止损”和“交易止损”混为一谈
- 默认先解释结构，再给交易结论
