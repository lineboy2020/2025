# 策略成功经验模板

> 用于沉淀每次回测、迭代、复盘得到的有效经验，逐步形成可复用、可验证、可升级的策略经验库。
>
> 使用原则：
> - 记录**事实**，不要只记录感觉
> - 区分**统计结论 / 策略假设 / 正式规则**
> - 每条经验都要写明**适用条件**与**证据来源**
> - 后续迭代时，只把“反复验证有效”的经验逐步提升为正式规则

---

# 一、使用方法

每次完成一轮回测、统计分析、策略迭代后，新增一条经验记录。

建议顺序：
1. 先写本次回测背景
2. 再写数据事实
3. 再写得出的经验结论
4. 最后写“下轮如何应用”

**注意：**
- 不要把所有发现都直接升级成正式规则
- 先进入“观察经验”或“试运行经验”
- 经过多轮验证后，再升级为“核心经验”

---

# 二、经验状态分级

## 1. 观察经验
- 有一定统计迹象
- 但样本不够或阶段不够完整
- 暂时不直接进入主策略

## 2. 试运行经验
- 已有较明确统计优势
- 可以在下一轮回测中适度纳入
- 但不能直接全量替换主规则

## 3. 正式经验
- 经过多轮、多阶段验证有效
- 可以正式写入主策略

## 4. 废弃经验
- 过去曾有效，但后续验证失效
- 保留记录，不再纳入现行策略

## 得出的经验结论
- 这次真正学到了什么？

示例：

## 经验适用条件
- 这条经验在什么条件下成立？
- 哪些环境下不适用？

可从以下角度写：
- 市场环境：强势 / 震荡 / 弱势
- 周期环境：月线做多 / 周线做多 / 日线试错
- 股票类型：趋势股 / 低位反转 / 高波动题材 / 大市值蓝筹
- 时间条件：尾盘 / 隔日 / 持有3-5天

## 对策略的影响判断
- [ ] 仅记录，不应用
- [ ] 下一轮小范围试运行
- [ ] 升级为正式规则
- [ ] 暂时搁置，等待更多样本
- [ ] 废弃旧规则，替换为新规则

## 下一轮如何应用
- 下一轮准备怎么用这条经验？
- 是加入过滤条件？
- 还是作为排序因子？
- 还是作为风险控制条件？

示例：
- 下一轮回测仅保留 B 类模板作为主样本
- 月线非 allow_long 环境下，不纳入日线1买
- 周线未确认上行段时，信号降级为观察池

## 风险与反例
- 这条经验可能在哪些情况下失效？
- 是否存在样本偏差、时间偏差、市场阶段偏差？

## 本轮结论摘要（一句话）
- 用一句最简洁的话总结本条经验

示例：
- “三周期策略中，周线强势回调1买是当前最值得保留的核心经验。”

# 五、经验库维护原则

## 原则 1：先证据，后结论
先有统计事实，再写经验，不要反过来。

## 原则 2：一次只迭代少数变量
不要一轮同时改买点、卖点、过滤、仓位，否则无法判断到底是哪项起作用。

## 原则 3：正式经验必须跨阶段有效
不是某一小段行情有效，就能进入主策略。

## 原则 4：保留失败经验
失败经验也要记录，避免以后重复犯错。

## 原则 5：经验库要持续升级
经验库不是笔记堆，而是策略系统的一部分。

## 经验标题
- 周线强势回调1买优先保留

- [ ] 正式经验

## 本次研究背景
- 对 B/C/D 三类样本进行分组统计，比较哪类更适合作为主买模型。

## 数据范围
- 市场范围：A股样本池
- 时间范围：近一年
- 样本数量：B=48, C=17, D=20
- 回测方式：分组统计

## 统计事实（只写事实）

- C类样本胜率 29.41%，平均收益 -2.428%
- D类样本胜率 35.00%，平均收益 -1.989%

## 得出的经验结论
- 当前最有效的经验是：周线强势回调1买明显优于其他两类样本。

## 经验适用条件
- 更适合周线已有上行结构的标的
- 不适合月线方向不清、周线未确认上行的标的

## 对策略的影响判断

- 下一轮优先只保留 B 类样本进入主买池，C/D 类先降级为观察样本。

## 风险与反例
- 可能受样本阶段影响，需要跨阶段继续验证。

## 本轮结论摘要（一句话）
- 周线强势回调1买，是当前最值得保留的三周期核心经验。
